# Frontciti Checklist

## Product Fit

- [ ] Register is correct: brand or product.
- [ ] Design matches the product category.
- [ ] Design matches the audience.
- [ ] Copy is specific.
- [ ] Main user goal is obvious.

## Visual Quality

- [ ] Clear hierarchy.
- [ ] Consistent spacing.
- [ ] Intentional typography.
- [ ] Intentional color.
- [ ] No generic AI-looking sections.
- [ ] Radius, border, shadow, and surface rules are consistent.
- [ ] Visual assets are real, generated, or clearly marked placeholders.

## UX

- [ ] Primary action is clear.
- [ ] Navigation is understandable.
- [ ] Forms are usable.
- [ ] Tables and lists have useful actions.
- [ ] Empty, loading, and error states exist where needed.
- [ ] UX copy explains action and recovery.

## Accessibility

- [ ] Semantic HTML.
- [ ] Keyboard navigation.
- [ ] Visible focus states.
- [ ] Labels for inputs.
- [ ] Contrast is acceptable.
- [ ] Reduced motion is supported.

## Responsive

- [ ] Mobile layout works.
- [ ] Tablet layout works.
- [ ] Desktop layout works.
- [ ] No overflow bugs.
- [ ] Touch targets are usable.
- [ ] Long text and translated labels do not break layout.

## Motion

- [ ] Motion has purpose.
- [ ] Motion does not block content.
- [ ] Animation cleanup is handled.
- [ ] Performance is acceptable.

## Production

- [ ] Code follows project conventions.
- [ ] No unnecessary dependencies.
- [ ] Components are maintainable.
- [ ] Edge cases are handled.
