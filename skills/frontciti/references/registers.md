# Registers

Frontciti chooses the design register before making visual decisions.

## Brand Register

Use for landing pages, marketing pages, campaigns, portfolios, long-form brand content, and pages where the impression is the product.

Priorities:

- Distinct point of view.
- Clear thesis above the fold.
- Specific copy and proof.
- Image or media system when the subject calls for it.
- Typography and color that feel chosen for this brand, not the category.
- One memorable move, supported by restraint elsewhere.

Brand can afford stronger pacing, committed color, art direction, and expressive type. It cannot afford generic templates, fake proof, or placeholder visuals.

## Product Register

Use for authenticated apps, dashboards, admin panels, tools, settings, data tables, forms, billing, onboarding, and client portals.

Priorities:

- Task completion.
- Predictable affordances.
- State visibility.
- Recoverability.
- Density control.
- Consistency with adjacent screens.
- Keyboard, focus, and error behavior.

Product UI can afford familiar patterns and tighter density. It should avoid decorative strangeness, page-load theatrics, and invented controls for standard tasks.

## Routing Questions

- Is the user trying to be convinced, or trying to complete a task?
- Is the page public-facing brand communication, or an operational surface?
- Would surprise help memorability, or would it slow the user down?
- Does the interface need to disappear into the workflow?

## Register Mistakes

- Treating dashboards like marketing pages.
- Treating landing pages like dense admin screens.
- Using brand display type in product labels.
- Using product density on a page that needs persuasion.
- Adding decorative motion to task flows.
