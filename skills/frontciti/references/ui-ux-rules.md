# UI/UX Rules

## Hierarchy

- Keep one primary action per view.
- Make secondary actions visible but quieter.
- Make destructive actions visually distinct.
- Make important alerts actionable.

## Layout

- Group related items.
- Keep scan paths predictable.
- Use whitespace to separate meaning.
- Avoid equal-weight sections unless they are truly equal.

## Interaction

- Give every interactive element visible feedback.
- Give forms labels, helper text, validation, error, and success states.
- Give modals focus management and escape behavior.
- Give menus and dropdowns keyboard support.

## States

Design for empty, loading, error, success, disabled, pending, and offline states where relevant.
