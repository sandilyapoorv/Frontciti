# Motion

Motion must serve the product.

## Use Motion For

- Showing cause and effect.
- Revealing hierarchy.
- Guiding attention.
- Making state changes understandable.
- Creating a memorable marketing moment.

## Avoid Motion For

- Blocking content.
- Hiding slow UI.
- Making dashboards harder to read.
- Adding visual noise.

## Technical Rules

- Prefer transform and opacity.
- Avoid layout-thrashing properties.
- Support reduced motion.
- Keep durations short and intentional.
- Use GSAP for complex timelines and scroll animation.
- Clean up animations in component lifecycles.
