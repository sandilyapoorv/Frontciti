# Anti-Slop Rules

Avoid designs that look like generic AI output.

## Common Patterns To Avoid

- Centered hero, huge gradient blob, vague headline.
- Three identical feature cards with generic icons.
- Random glass panels.
- Purple and blue gradients by default.
- Oversized rounded rectangles everywhere.
- Fake app screenshots.
- Repeated stock icons.
- Vague copy such as "seamless," "powerful," "unlock," and "transform."
- No real workflow.
- No states.
- No mobile thought.
- No accessibility.

## Better Approach

- Start from the product's real workflow.
- Use specific copy and proof.
- Build hierarchy from user intent.
- Make visual choices explainable.
- Use fewer, stronger design ideas.
