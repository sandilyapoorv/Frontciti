# /frontciti audit

## Purpose

Critique a frontend for visual design, UX, accessibility, responsiveness, performance, motion, and production readiness.

## Use When

Use when the user asks for review, critique, scoring, design QA, UX audit, accessibility audit, or improvement priorities.

## Inputs To Inspect

Layout, hierarchy, type, color, spacing, copy, navigation, interaction states, forms, tables, charts, mobile behavior, accessibility, performance, motion, and error states.

## Workflow

1. Inspect the UI and code paths that produce it.
2. Identify what the interface is trying to accomplish.
3. Separate objective usability problems from taste preferences.
4. Rank issues by user impact and implementation leverage.
5. Recommend concrete fixes.

## Output Expectations

Provide overall score, what works, biggest issues, priority fixes, specific implementation suggestions, quick wins, and high-impact opportunities.

## Anti-Patterns

- Vague feedback such as "make it cleaner."
- Equal priority for every issue.
- Critique that ignores the product context.
- Suggestions that require a full rewrite when small fixes would work.

## Acceptance Checklist

- Feedback is specific.
- Issues are prioritized.
- Fixes are actionable.
- The audit covers both UX and implementation risks.
