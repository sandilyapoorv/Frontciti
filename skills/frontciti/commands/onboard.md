# /frontciti onboard

## Purpose

Design first-run flows, activation paths, guided empty states, and progressive onboarding.

## Use When

Use when new users need help understanding what to do first, how to reach value, or how to recover from an empty product state.

## Inputs To Inspect

User roles, first-run state, setup requirements, empty states, permissions, sample data, onboarding steps, activation metric, and product workflows.

## Workflow

1. Identify the first meaningful user outcome.
2. Remove unnecessary setup before that outcome.
3. Turn empty states into guided action.
4. Use progressive disclosure instead of a giant tour.
5. Provide sample data, templates, checklists, or inline coaching where useful.
6. Include skip, back, and recovery paths.

## Output Expectations

Users should know what to do next and reach product value with minimal confusion.

## Anti-Patterns

- Forced multi-step tours.
- Empty dashboards with no guidance.
- Asking for too much setup too early.
- Coach marks that cover the UI.

## Acceptance Checklist

- First meaningful action is obvious.
- Empty states guide action.
- Onboarding is skippable or recoverable.
- Progress and system state are clear.
