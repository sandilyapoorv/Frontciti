# /frontciti redesign

## Purpose

Improve an existing UI without blindly replacing it.

## Use When

Use when a screen, page, flow, or component already exists and should become clearer, more polished, more responsive, or more product-specific.

## Inputs To Inspect

Current UI, code, screenshots, route context, data shape, user flow, copy, styles, and known product constraints.

## Workflow

1. Identify what the screen is trying to accomplish.
2. Preserve working flows and useful brand equity.
3. List top issues in hierarchy, spacing, type, layout, copy, accessibility, responsiveness, and states.
4. Define a sharper design direction.
5. Implement improvements incrementally.
6. Compare the result against the original user goal.

## Output Expectations

The redesign should improve clarity, product fit, polish, and usability while preserving functionality.

## Anti-Patterns

- Replacing a working UI with a generic template.
- Removing behavior that users depend on.
- Decorating before clarifying hierarchy.
- Bolder visuals that reduce credibility.

## Acceptance Checklist

- Existing functionality remains.
- UI is clearer and more polished.
- Brand and product fit improve.
- Mobile behavior improves.
- Accessibility does not regress.
