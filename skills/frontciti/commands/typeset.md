# /frontciti typeset

## Purpose

Improve typography hierarchy, font choices, scale, measure, and text rhythm.

## Use When

Use when a UI feels generic, hard to read, cramped, visually flat, or typographically inconsistent.

## Inputs To Inspect

Font stack, loaded weights, headings, body text, labels, data, line lengths, line heights, letter spacing, responsive type, and brand/product register.

## Workflow

1. Decide whether the surface needs brand personality or product-task familiarity.
2. Audit font families, type scale, weights, line length, line height, tracking, and consistency.
3. Prefer one well-tuned family for product UI; use contrast thoughtfully for brand surfaces.
4. Build a role-based scale for caption, secondary, body, subheading, heading, and display.
5. Keep body text readable and zoom-safe.

## Output Expectations

Typography should carry hierarchy clearly, fit the register, and avoid default-font sameness where personality matters.

## Anti-Patterns

- Inter/Roboto/system fallback as reflex on brand surfaces.
- Too many font families.
- Sizes that are too close together.
- Display text that overflows or clips.
- Body text below 16px.

## Acceptance Checklist

- Hierarchy is instantly legible.
- Body measure is comfortable.
- Font choices fit the product.
- Type remains responsive and accessible.
