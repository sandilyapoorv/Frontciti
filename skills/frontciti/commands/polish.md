# /frontciti polish

## Purpose

Refine an existing UI so it feels finished, coherent, premium, and production-ready.

## Use When

Use when the structure is mostly right but the interface feels rough, inconsistent, generic, crowded, or unfinished.

## Inputs To Inspect

Spacing, type scale, colors, borders, radii, shadows, icons, copy, button hierarchy, states, transitions, and responsive details.

## Workflow

1. Preserve app logic and broad structure.
2. Tighten hierarchy and scan paths.
3. Normalize spacing, type, borders, radii, and elevation.
4. Improve copy, labels, and action hierarchy.
5. Add missing states and microinteractions.
6. Test responsive details and overflow.

## Output Expectations

Small changes should compound into a more complete, credible interface.

## Anti-Patterns

- Over-redesigning when refinement was requested.
- Adding visual noise to create "premium" feel.
- Changing core behavior without need.
- Polishing only desktop.

## Acceptance Checklist

- UI feels more complete.
- Hierarchy is clearer.
- Spacing is consistent.
- Interactions have feedback.
- No unnecessary clutter was added.
