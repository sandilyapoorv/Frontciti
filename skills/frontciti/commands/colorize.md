# /frontciti colorize

## Purpose

Improve color strategy, palette, contrast, theming, and semantic color usage.

## Use When

Use when a UI feels flat, grayscale, washed out, overly timid, inaccessible, or semantically unclear.

## Inputs To Inspect

Brand colors, token files, CSS variables, Tailwind config, contrast, dark mode, semantic state colors, charts, and existing surface colors.

## Workflow

1. Identify register and color strategy: restrained, committed, full palette, or drenched.
2. Preserve committed brand colors before inventing new ones.
3. Use color for hierarchy, meaning, wayfinding, state, or brand voice.
4. Build or refine primitive, semantic, and component tokens.
5. Verify contrast and color-blind resilience.

## Output Expectations

The result should use color intentionally and consistently, with accessible contrast and clear semantic roles.

## Anti-Patterns

- Random accent colors.
- Default purple-blue gradients.
- Gray text on colored backgrounds.
- Color as the only state indicator.
- Hard-coded colors where tokens exist.

## Acceptance Checklist

- Palette has a strategy.
- Color meanings are consistent.
- Contrast is acceptable.
- Theming remains coherent.
