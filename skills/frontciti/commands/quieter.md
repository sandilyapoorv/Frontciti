# /frontciti quieter

## Purpose

Make a design calmer, cleaner, more enterprise, more minimal, or more trustworthy.

## Use When

Use when a UI feels noisy, over-styled, too decorative, too consumer-like, or too visually competitive for the product context.

## Inputs To Inspect

Color intensity, shadows, borders, gradients, motion, competing CTAs, copy density, hierarchy, spacing, data density, and enterprise trust cues.

## Workflow

1. Preserve important actions and hierarchy.
2. Reduce excess color, shadows, borders, gradients, and motion.
3. Tighten copy and remove duplicate calls to action.
4. Improve neutral hierarchy, spacing, data clarity, and trust signals.
5. Confirm the result still feels designed, not stripped bare.

## Output Expectations

The result should feel calmer and more credible while keeping primary actions visible.

## Anti-Patterns

- Removing all personality.
- Making the primary action hard to find.
- Creating a flat gray interface with no hierarchy.
- Reducing information that users need for decisions.

## Acceptance Checklist

- Design feels calmer.
- Hierarchy remains clear.
- Important actions are still visible.
- Product feels more credible.
