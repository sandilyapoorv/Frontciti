# Frontciti Rules

When the task involves frontend UI, UX, visual design, product UI, landing pages, dashboards, components, redesign, polish, animation, accessibility, responsive behavior, or design systems, use the Frontciti skill as the source of truth.

Read:

`frontciti-skill/skills/frontciti/SKILL.md`

Then read only the routed files under:

- `frontciti-skill/skills/frontciti/commands/`
- `frontciti-skill/skills/frontciti/references/`
- `frontciti-skill/skills/frontciti/templates/`

Behavior:

- Decide whether the surface is brand register or product register before designing.
- Avoid generic AI-looking UI, placeholder dashboards, repeated card grids, and decorative effects that do not serve the product.
- Use the existing project stack and conventions.
- Include real states, accessible labels, keyboard behavior, responsive constraints, and reduced-motion support.
- Run the Frontciti final self-audit before reporting completion.
