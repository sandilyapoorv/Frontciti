# Frontciti Instructions

Use Frontciti for frontend design and implementation work: landing pages, SaaS UI, dashboards, product surfaces, components, redesigns, critique, polish, accessibility, responsive behavior, UX copy, and motion.

Read this skill before acting on matching frontend tasks:

`frontciti-skill/skills/frontciti/SKILL.md`

Use progressive disclosure:

- Read `SKILL.md` first.
- If the user names a `/frontciti` command, read the matching file in `commands/`.
- Load only the references relevant to the current task.

Implementation expectations:

- Match the existing codebase stack and design system.
- Prefer semantic HTML, accessible controls, responsive constraints, and real UI states.
- Use purposeful motion and support reduced motion.
- Check the final UI against the Frontciti self-audit before completion.
